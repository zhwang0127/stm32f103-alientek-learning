# 🌙 智能小夜灯 — 项目讲解

> 基于 STM32F103C8T6 + BEE BLOCK 积木模块 | 2026-01-15

---

## 一、项目概述

一个根据环境光照自动调节亮度的小夜灯。核心功能只有三句话：

- **环境暗 → 灯自动亮（暖色温）**
- **环境亮 → 灯自动灭**
- **按一下键 → 切换模式（自动 / 常亮 / 呼吸）**

虽然功能简单，但它串起了 STM32 开发中最核心的四个外设：**ADC、TIM(PWM)、I2C、GPIO**。做完这个项目，你就掌握了嵌入式开发的基本套路。

```

│                   M48Z-M3 最小系统板                      │
│                   STM32F103C8T6                         │
│                                                         │
│  ┌──────────┐   ┌──────────┐   ┌──────────────────┐    │
│  │ 光敏传感器 │   │ OLED 屏   │   │   RGB LED        │    │

│  │           │   │          │   │                  │    │
│  │ AO──PA0   │   │ SCL─PA2  │   │ R──PA6 (TIM3_CH1)│    │
│  │ (ADC1_CH0)│   │ SDA─PA3  │   │ G──PA7 (TIM3_CH2)│    │
│  │           │   │ (软件I2C) │   │ B──PB0 (TIM3_CH3)│    │
│  └──────────┘   └──────────┘   └──────────────────┘    │
│                                                         │
│  KEY0(PC13) ── 模式切换按键                              │
│  USART1(PA9/PA10) ── 串口调试输出                        │
└─────────────────────────────────────────────────────────┘
```

### 引脚分配表

| 引脚 | 外设 | 功能 | 备注 |
|:---:|------|------|------|
| PA0 | ADC1_CH0 | 读取光敏传感器模拟值 | 原为 PA2，因 OLED 冲突改到 PA0 |
| PA2 | GPIO 推挽 | OLED SCL (软件 I2C 时钟) | 与风扇调速器项目一致 |
| PA3 | GPIO 开漏 | OLED SDA (软件 I2C 数据) | 与风扇调速器项目一致 |
| PA6 | TIM3_CH1 | RGB LED 红色通道 PWM | 72kHz, 1000 级精度 |
| PA7 | TIM3_CH2 | RGB LED 绿色通道 PWM | 同上 |
| PB0 | TIM3_CH3 | RGB LED 蓝色通道 PWM | 同上 |
| PC13 | GPIO 输入 | KEY0 模式切换按钮 | 板载按键 |
| PA9 | USART1_TX | 串口调试输出 | 115200bps |
| PA10 | USART1_RX | 串口调试输入 | 未使用 |

### 为什么选择这个引脚分配？

这个分配解决了两个关键冲突：

| 冲突 | 说明 | 解决 |
|---|---|---|
| ADC vs OLED | 光敏传感器默认用 PA2(AO) + PA3(DO)，OLED 用 PA2(SCL) + PA3(SDA) | 光敏 AO 移到 PA0，DO 弃用 |
| WK_UP vs ADC | 板载 WK_UP 按键在 PA0，与光敏 ADC 冲突 | 移除 WK_UP 初始化，只用 KEY0(PC13) |

## 四、关键算法详解

### 4.1 光敏值到亮度的映射

光敏传感器 `atk_light_get_val()` 返回值：**0 = 环境最亮，100 = 环境最暗**。

```
        返回值
        100 ┤                        ┌─ 全亮区 (light_val ≤ 25)
            │                     ╱
            │                  ╱
            │               ╱
            │            ╱  ← 线性映射区 (25~80)
            │         ╱
            │      ╱
          0 ┤────┘──────────────────
            0    25              80   100
            ↑                    ↑
         最暗                  最亮
         LED最亮              LED全灭

          └─ 全灭区 (light_val ≥ 80)
```

**代码实现：**

```c
if (light_val <= 25)
    target = 100;           // 很暗 → 灯全亮
else if (light_val >= 80)
    target = 0;             // 够亮 → 关灯
else
    target = (light_val - 25) * 100 / 55;   // 线性映射
```

### 4.2 平滑过渡（消除突兀跳动）

不做平滑 → 灯光会因 ADC 噪声闪烁。每帧只变化 ±1，视觉上丝滑过渡：

```c
void smooth_to(uint8_t target, uint8_t *current)
{
    if (target > *current) (*current)++;    // 渐亮
    if (target < *current) (*current)--;    // 渐暗
}
```

### 4.3 暖色温映射

人眼对暖色光（低色温）更舒适。RGB 比例固定为 `1 : 0.5 : 0.25`：

```c
r = brightness * 999 / 100;          // 亮度 → PWM 值
g = r / 2;                           // 绿 = 红的 50%
b = r / 4;                           // 蓝 = 红的 25%
```

输出效果：暗环境 → 温暖橙光，模拟烛光/白炽灯。

### 4.4 呼吸灯算法

简单的递增递减循环，不需要三角函数：

```c
val += dir;                  // dir = ±5
if (val >= 999) dir = -5;    // 触顶反转
if (val == 0)   dir = +5;    // 触底反转
// 周期 ≈ 999/5 × 2 × 25ms = 10 秒一个呼吸周期
```

## 五、外设配置一览

### 5.1 ADC — 光敏传感器读取

| 参数 | 值 |
|---|---|
| ADC 模块 | ADC1 |
| 通道 | CH0 (PA0) |
| 时钟 | PCLK2/6 = 12MHz |
| 分辨率 | 12bit (0~4095) |
| 采样时间 | 239.5 周期 ≈ 21μs |
| 读取方式 | 软件触发 + 轮询 |
| 均值滤波 | 10 次取平均 |

### 5.2 TIM3 PWM — RGB LED 驱动

| 参数 | 值 |
|---|---|
| 定时器 | TIM3 (通用定时器) |
| 时钟源 | APB1×2 = 72MHz |
| 预分频 PSC | 0 (不分频) |
| 自动重载 ARR | 999 |
| PWM 频率 | 72MHz / 1000 = **72kHz** |
| 精度 | 1000 级 (0~999) |
| 通道 | CH1(PA6)=R, CH2(PA7)=G, CH3(PB0)=B |
| 极性 | PWM 模式1，低电平有效 |

> **为什么选 72kHz？** 远超人耳可听范围 (20kHz)，无电磁啸叫；远超人眼闪烁感知 (>100Hz)，完全无频闪感知。

### 5.3 软件 I2C — OLED 通信

| 参数 | 值 |
|---|---|
| 引脚 | SCL=PA2, SDA=PA3 |
| 速率 | ~250kHz |
| 设备地址 | 0x78 |
| 控制器 | SSD1306 |
| 分辨率 | 128×64 |
| 显存 | g_atk_oled_gram[128][8] (1KB) |

## 六、三种工作模式

| 模式 | 按键次数 | OLED 显示 | 行为 |
|---|---|---|---|
| **Auto** | 0 (默认) | `Mode: Auto` | 环境暗→灯渐亮，环境亮→灯渐灭 |
| **Const** | 按 1 次 | `Mode: Const` | 固定 50% 暖光，不受环境光影响 |
| **Breath** | 按 2 次 | `Mode: Breath` | 独立呼吸灯，每个周期约 10 秒 |

## 十、常见问题

**Q: OLED 不显示？**  
A: 检查 PA2/PA3 接线，OLED 地址是否正确（0x78）。可先用 `风扇调速器` 工程验证 OLED 硬件正常。

**Q: 光敏传感器读数不变？**  
A: 检查 PA0 是否连接了光敏模块的 AO 输出端（不是 DO）。用手遮挡应有明显变化。

**Q: RGB LED 不亮？**  
A: RGB LED 是共阳极还是共阴极？本项目配置为低电平有效（OCPolarity=LOW），如果反了需要改为 HIGH 或硬件对调 VCC/GND。

**Q: 编译报错找不到头文件？**  
A: 确保 Include Paths 包含所有 BSP 子目录。工程文件已配好，直接打开 `.uvprojx` 即可。

**Q: 呼吸灯太慢/太快？**  
A: 修改 `mode_breath()` 里的 `dir = ±5` 值——值越大呼吸越快（范围 1~50）。
